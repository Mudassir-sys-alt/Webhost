Role-Based Access Control Test

Test Accounts:

Admin: admin@loadshare.com / admin123

Mudassir (Admin): mudassir@loadshare.net / mudassir123

Manager: manager1@loadshare.com / manager123

User: user1@loadshare.com / user123

Admin Role Permissions

Users with admin role (including mudassir@loadshare.net) have full access to all features.

✅ Add New Bikes
✅ Edit Bikes
✅ Delete Bikes
✅ Export Data
✅ View All Inventory
✅ Search & Filter

Manager Role Permissions

Users with manager role can add, edit, and export but cannot delete.

✅ Add New Bikes
✅ Edit Bikes
❌ Delete Bikes
✅ Export Data
✅ View All Inventory
✅ Search & Filter

User Role Permissions

Users with user role can only view and search the inventory.

❌ Add New Bikes
❌ Edit Bikes
❌ Delete Bikes
❌ Export Data
✅ View All Inventory
✅ Search & Filter

How to Test

  1. Open the main dashboard page (dashboard.html)
  2. You will see a login modal automatically
  3. Login with any of the test accounts above
  4. Navigate to "Transactions" → "Bikes Inventory"
  5. Try to use the Add, Edit, Delete, and Export buttons
  6. Notice how the permissions change based on your role
  7. Logout and try with different accounts to see the differences

Technical Implementation

  • User Management: user-management.js handles authentication and role checking
  • Permission Checks: Each action checks userManager.canEdit(), canDelete(), etc.
  • UI Updates: Buttons are disabled/enabled based on permissions
  • Visual Feedback: Role badges and access denied messages
  • Session Management: Uses localStorage to persist login state
